0. Análisis Técnico Frontal: ¿Cómo funciona JAVA universalmente?

El principal motor funcional de este material educativo es diseccionar la robusta arquitectura subyacente de Java. A diferencia del software regular compilado en C o Ensamblador, Java funciona con la magia dual del Bytecode.

Representación de Compilador transformando código abstracto a motores universales

Etapas obligatorias del Intérprete:

  • 1. El Origen (.java): Lugar físico donde los programadores estructuran las clases y métodos usando sintaxis de alto nivel.
  • 2. El Compilador (javac): Centinela que valida la sintaxis y convierte el código humano a jeroglífico de bajo nivel dominado universalmente: El Bytecode (.class). No corre en la computadora, es neutro.
  • 3. JVM (Máquina Virtual de Java): Software intérprete preinstalado en dispositivos cliente (Windows, Mac, Linux). Consume el Bytecode on-the-fly e inyecta la memoria RAM nativa permitiendo el dogma universal sagrado: "Escribe tu código una vez, domínalo en cualquier lado (WORA)".

1. Tipos de Datos (Primitivos y Complejos)

📄 DataTypesCLI.java | DataTypesGUI.java

Este proyecto está diseñado metodológicamente para enseñar fundamentos paralelos estructurados mediante ejemplos por consola (CLI) y pantallas (Swing GUI).

Objetivo Práctico

Demuestra los tipos primitivos nativos disponibles bajo reserva de memoria estricta en Java (byte, short, int, long, float, double, boolean, char) y los referenciados como (String).

Ejecución y Resultados

El sistema en vivo imprime descriptivamente los diferentes tamaños de bit en memoria ram e instanciamientos físicos de variables positivas máximas simulando ser inyectadas un paso a la vez.

2. Operaciones Matemáticas

📄 MathOpsCLI.java | MathOpsGUI.java

Objetivo Práctico

Exposición elemental sobre el uso de operadores aritméticos (+, -, *, /, %) y funciones complementarias de la librería estandarizada java.lang.Math (como raíces cuadradas y potencias numéricas).

Ejecución y Resultados

A través de la captura interactiva del usuario (Ej. Numero 1 y 2), el procesador emite el resultado calculadamente blindando crasheos inminentes usando validaciones protectoras frente a divisiones asintóticas computacionales por cero.

3. Estructuras de Datos Universales

📄 DataStructuresCLI.java | DataStructuresGUI.java

Objetivo Práctico

Introducción rotunda a los contenedores de modelado y estructuración dinámica estandarizados en Java Collections Framework. Análisis minucioso de listas enlazadas en RAM (`ArrayList`) y matrices asociativas veloces (Llave-Valor abstracta) mediante diccionarios (`HashMap`).

4. Estructuras Algorítmicas Básicas

📄 AlgorithmicStructuresCLI.java

Objetivo Práctico

Conjunta a las 3 ramificaciones fundamentales de las máquinas de Turing: Asignación, Decisiones Lógicas if/else/switch, y bucles cíclicos for/while. Las simulamos de manera aplicada haciendo iteraciones a nivel "Ingeniero de Sistemas" procesando rastreo de IPs y métricas límite de Servidores Web saturados ficticios.

5. Funciones, Métodos y Recursión

📄 MethodsAndFunctionsCLI.java

Objetivo Práctico

Demuestra la construcción de módulos reciclables public static void/return. Se ejemplifica recibiendo un Email y aplicando métodos estelares de ofuscamiento para censurarlo en terminales, culminando en la Recursividad Pura (funciones numéricas que se llaman a sí mismas emulando un stack arquitectónico hasta resolver factoriales recursivos de orden O(n)).

6. Programación Orientada a Objetos (POO)

📄 OOPCLI.java | OOPGUI.java

Objetivo Práctico

Estudio anatómico del molde maestro vs las instancias clónicas en la RAM:

  • Clase Base (Empleado): Inyecta ADN variable (nombre) y la acción estática genérica devolviendo String: "Trabajando en general...".
  • Sub-Especie (Ingeniero): Extiende inminentemente extends Empleado, incautando gratuitamente todas las reglas lógicas del Padre y ahorrando código.
  • Polimorfismo de Mutación (@Override): El Ingeniero sobreescribe rebeldiamente la función trabajar() comportándose autónomamente como "Ingeniero programando en Java". Dos entes, un mismo comando biológico, rutinas totalmente separadas.

7. Manejo Probabilizado de Excepciones

📄 ExceptionsCLI.java | ExceptionsGUI.java

Objetivo Práctico

Inmuniza formalmente fallos sistémicos de compilador ("Crashes"). Acorazando lógicas amenazadas incrustadas en try {} salvaguardadas militarmente bajo controles de recolección de falla catch(Exception e). Acomulando todo el flujo seguro bajo un portal determinístico irrompible finally {}.

8. Arreglos Fijos Absolutos Bi/Tridimensionales

📄 FixedSizeArraysCLI.java

Objetivo Práctico

Reserva cuadriculada plana no extenible (int[] x = new int[5]). Se navega mediante Sorts ascendentes e ínices cruzados, abriéndose puertas para explorar matrices estilo Excel 2x2x2 bidimensionales de complejidad O(1) resolutivas hasta deducir métricas estadísticas universales como Desviaciones Estándar poblacionales.

9. Modificadores de Contenedores de Fábrica (LIFO y FIFO)

📄 StandardContainersCLI.java | StandardContainersGUI.java

Objetivo Práctico

El código recrea las dinámicas reales dictadas de colas asíncronas de la universidad:

  • Stack de Memoria (Pilas LIFO): Un Navegador de Internet con botón "Volver Atrás" construido mediante ventanas modales asíncronas de JOptionPane de Push y Pop alocadas dinámicamente.
  • Gestor FIFO: Descargas de navegadores secuenciales en Queue iterativa con Poll pasivo y el Scheduler brutal del OS sobrepasando prioridades de hilo del procesador PriorityQueue.

10. Inyecciones Estéticas de Interfaz (Modo Oscuro)

📄 GraphicalCustomizationGUI.java

Objetivo Práctico

Manipusión paramétrica en Runtime sobre el esqueleto UIManager de Swing; forzando componentes rígidos de C++ en Java a traslucirse al paradigma visual de estética 'Dark Theme' dictando paletas RGB formales (`setForeground(Color)`).

11. Contenedores Exóticos No Lineales (Sets y Maps de Tablas Hash)

📄 NonLinearContainersCLI.java

Objetivo Práctico

Aplicación contundente de tablas Hash que no son iterables pero devoran índices O(1). Usa simulaciones de Cortafuegos rechazando IPs y subredes quemadas asíncronamente con un escudo defensivo lógico de Clave duplicada a expensas de la memoria HashSet estricta.

12. Adopciones del Java Moderno (Registros, PatternMatching)

📄 ModernJavaCLI.java

Objetivo Práctico

Refactorización de arquetipos hacia las mecánicas pos-JDK 16. Destruye los pesados Getters/Setters anidando la inmutabilidad veloz de las record Variables(int a) {}, elude casteo sucio aplicando instanceOf Match, e inhabilita engorrosos for-loops abrazando Tuberías Algorítmicas Lambda para filtrar nodos de Servidor caídos con stream().filter().forEach().
