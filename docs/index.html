0. Fundamentos de Java: Arquitectura, Historia y Sintaxis

El principal motor funcional de este material educativo es diseccionar la robusta arquitectura subyacente de Java. A diferencia del software regular compilado en C o Ensamblador, Java funciona con la magia dual del Bytecode.

Representación del Compilador transformando código abstracto universalmente


📖 1. Un Poco de Historia

Java nació en 1991 como el proyecto secreto "Green Project" bajo la tutela de James Gosling en Sun Microsystems. En sus inicios se llamó Oak debido a un roble que Gosling veía por su ventana.

Su lema fundacional fue crear un código neutro para programar electrodomésticos, pero terminó dominando la Internet mundial en 1995 al crear el mantra universal: Write Once, Run Anywhere (WORA).


⚙ 2. Etapas obligatorias del Intérprete (Arquitectura):

  • El Origen (.java): Lugar físico donde programamos las clases usando sintaxis de alto nivel.
  • El Compilador (javac): Centinela que valida la sintaxis y convierte el código humano al lenguaje intermedio: El Bytecode (.class). Este código no corre en hardware, es completamente neutro.
  • JVM (Máquina Virtual de Java): Un software especializado (parte del JRE) que consume el Bytecode on-the-fly y lo traduce al procesador local (Windows, Mac, Linux) instantáneamente.

⌨ 3. Didáctica de Sintaxis Elemental

Java es de la familia de C++. Obliga al desarrollador a ser disciplinado. La tipificación es Fuerte y es 100% Orientado a Objetos. Nunca verás una variable suelta flotando en el vacío; todo habita dentro de bloques class.


public class HolaMundo {
    // Método estático principal: El punto de arranque de la JVM
    public static void main(String[] args) {
        // Tipado fuerte: Declaras el INT antes de usarlo
        int edad = 30; 
        System.out.println("Sintaxis perfecta. Edad: " + edad);
    }
}

📦 4. Gestores Corporativos (Maven vs Gradle)

Añadir repositorios arrastrando binarios .jar a mano es cosa del pasado. Hoy en día, utilizamos orquestadores automatizados (CI/CD) para resolver árboles de dependencias masivas.

  • Apache MAVEN: El patriarca. Utiliza un archivo rígido en formato XML (pom.xml). Sólido, predecible, ideal para backend empresarial estricto.
  • GRADLE: El contendiente moderno. Usa scripts ágiles en Kotlin o Groovy (build.gradle). Compila de forma modular y asíncrona, haciéndolo brutalmente rápido (Es el motor obligatorio de Android Studio).

1. Tipos de Datos Básicos (Memoria RAM)

📄 DataTypesCLI.java | DataTypesGUI.java

Objetivo Práctico

Conocer cómo Java estampa la información del mundo real en la memoria RAM utilizando sistemas de clasificación fuertemente tipados. En lugar de estudiar todos los límites numéricos exhaustivos, nos enfocamos en los 4 tipos más útiles del mercado para crear rápidamente un Perfil de Usuario:

  • String (Cadenas de texto. Ej: Nombre)
  • int (Enteros exactos matemáticos. Ej: Edad)
  • double (Números decimales precisos. Ej: Estatura)
  • boolean (Lógica binaria 1/0. Ej: Activo/Inactivo)

Ejecución y Resultados

El sistema en vivo imprime descriptivamente la inyección de estas 4 variables construyendo un perfil humano, evaluando un paso a la vez el impacto en la Memoria RAM.

📄 MethodsAndFunctionsCLI.java

Objetivo Práctico

Se ejemplifica recibiendo un Email y aplicando métodos estelares de ofuscamiento para censurarlo en terminales, culminando en la Recursividad Pura (funciones numéricas que se llaman a sí mismas emulando un stack arquitectónico hasta resolver factoriales recursivos de orden O(n)).

6. Programación Orientada a Objetos (POO)

📄 OOPCLI.java | OOPGUI.java

Objetivo Práctico

Estudio anatómico del molde maestro vs las instancias clónicas en la RAM:

  • Clase Base (Empleado): Inyecta ADN variable (nombre) y la acción estática genérica devolviendo String: "Trabajando en general...".
  • Sub-Especie (Ingeniero): Extiende inminentemente extends Empleado, incautando gratuitamente todas las reglas lógicas del Padre y ahorrando código.
  • Polimorfismo de Mutación (@Override): El Ingeniero sobreescribe rebeldiamente la función trabajar() comportándose autónomamente como "Ingeniero programando en Java". Dos entes, un mismo comando biológico, rutinas totalmente separadas.

7. Manejo Probabilizado de Excepciones

📄 ExceptionsCLI.java | ExceptionsGUI.java

Objetivo Práctico

Inmuniza formalmente fallos sistémicos de compilador ("Crashes"). Acorazando lógicas amenazadas incrustadas en try {} salvaguardadas militarmente bajo controles de recolección de falla catch(Exception e). Acomulando todo el flujo seguro bajo un portal determinístico irrompible finally {}.

8. Arreglos Fijos Absolutos Bi/Tridimensionales

📄 FixedSizeArraysCLI.java

Objetivo Práctico

Reserva cuadriculada plana no extenible (int[] x = new int[5]). Se navega mediante Sorts ascendentes e ínices cruzados, abriéndose puertas para explorar matrices estilo Excel 2x2x2 bidimensionales de complejidad O(1) resolutivas hasta deducir métricas estadísticas universales como Desviaciones Estándar poblacionales.

9. Modificadores de Contenedores de Fábrica (LIFO y FIFO)

📄 StandardContainersCLI.java | StandardContainersGUI.java

Objetivo Práctico

El código recrea las dinámicas reales dictadas de colas asíncronas de la universidad:

  • Stack de Memoria (Pilas LIFO): Un Navegador de Internet con botón "Volver Atrás" construido mediante ventanas modales asíncronas de JOptionPane de Push y Pop alocadas dinámicamente.
  • Gestor FIFO: Descargas de navegadores secuenciales en Queue iterativa con Poll pasivo y el Scheduler brutal del OS sobrepasando prioridades de hilo del procesador PriorityQueue.

11. Colecciones Avanzadas (Sets y Maps)

📄 NonLinearContainersCLI.java | NonLinearContainersGUI.java

Objetivo Práctico

Estructuras que rompen la linealidad tradicional de las listas. Muy útiles en la vida diaria:

  • Conjuntos (HashSet): Bloquean automáticamente los duplicados matemáticos. Ejemplo: Una lista de correos electrónicos únicos o IDs.
  • Mapas (HashMap): Diccionarios en pareja (Clave -> Valor). Ejemplo: Una agenda que asocia tu Nombre con tu Teléfono O(1).

12. Adopciones de Java Moderno (Records y Streams)

📄 ModernJavaCLI.java | ModernJavaGUI.java

Objetivo Práctico

La programación se ha vuelto más limpia. Aprende cómo Java moderno acorta el código clásico:

  • Records: Reemplaza decenas de líneas de Getters y Setters para declarar moldes básicos de datos (Ej: un molde Person con Nombre y Edad).
  • Tuberías con Streams: En vez de crear bucles For difíciles, usa tuberías lógicas legibles (.filter().forEach()) para por ejemplo, filtrar personas mayores de 18 años instantáneamente.
